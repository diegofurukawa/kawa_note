## File Descriptions

### Public Assets

| File | Purpose | Format | Size | Notes |
|------|---------|--------|------|-------|
| `logo.svg` | Primary logo (light backgrounds) | SVG | 200x200 | Scalable, no raster |
| `logo-dark.svg` | Dark theme variant | SVG | 200x200 | Light colors on dark |
| `icon.svg` | Icon-only version | SVG | 64x64 | Favicon-ready |
| `favicon.ico` | Browser favicon | ICO | Multi-size | 16x16, 32x32, 64x64 |
| `og-image.png` | Social sharing image | PNG | 1200x630 | <1MB, optimized |
| `og-image-square.png` | Square variant | PNG | 800x800 | For Pinterest, etc. |
| `manifest.json` | PWA manifest | JSON | — | App metadata |
| `robots.txt` | SEO robots directive | TXT | — | Crawl instructions |

### SEO Metadata Files

#### `metadata.ts`
- Core meta tags (title, description, robots, viewport, charset)
- Theme color
- Apple mobile web app settings
- Exported as `seoMetadata` object + `generateMetaTags()` function

#### `openGraph.ts`
- Open Graph tags for social sharing
- Image metadata (1200x630 recommended)
- Alternative images for different contexts
- Exported as `openGraphMetadata` object + `generateOpenGraphTags()` function

#### `twitterCard.ts`
- Twitter Card configuration
- Card type: summary_large_image
- Image alt text
- Exported as `twitterCardMetadata` object + `generateTwitterCardTags()` function

#### `jsonld.ts`
- JSON-LD schemas for search engines
- SoftwareApplication schema (app metadata, ratings, features)
- Organization schema (company info, contact)
- Website schema (search action)
- Breadcrumb schema (navigation)
- Exported as generator functions

### React Components

#### `MetaHead.jsx`
- Injects meta tags into document head
- Accepts overrides for dynamic pages
- Usage: `<MetaHead title="Custom Title" description="Custom description" />`

#### `StructuredData.jsx`
- Injects JSON-LD schemas
- Accepts schema type (SoftwareApplication, Organization, etc.)
- Usage: `<StructuredData schema={generateSoftwareApplicationSchema()} />`

```html
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

```jsx
    </>

```

## Scalability Rationale

1. **Centralized Configuration:** All branding/SEO data in one place (easy to update)
2. **Reusable Functions:** Generate tags dynamically (supports multiple pages)
3. **Type Safety:** TypeScript definitions (prevents errors)
4. **Component-Based:** React components (easy to integrate)
5. **Asset Organization:** Public assets separate from code (easy to manage)
6. **Future-Proof:** Structure supports ecosystem expansion (Kawa Graph, Kawa Teams, etc.)

## Maintenance

- **Monthly:** Review SEO metrics, update OG images if needed
- **Quarterly:** Audit branding consistency, update color/typography if needed
- **Annually:** Security audit, brand perception survey, competitive analysis
- **As Needed:** Update metadata for new features/products
