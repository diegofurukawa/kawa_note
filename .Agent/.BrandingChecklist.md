## 2. Cross-Environment Usage Rules

### Web Application
- [ ] Logo placed in header (left-aligned, 32-40px height)
- [ ] Favicon: icon.svg (16x16, 32x32, 64x64 variants)
- [ ] Theme color: #0f766e (teal)
- [ ] Dark mode: Use logo-dark.svg with light backgrounds

### Landing Page
- [ ] Hero section: Primary logo + tagline
- [ ] Feature cards: Use icon.svg as visual anchors
- [ ] CTA buttons: Consistent color (primary teal)
- [ ] Footer: Logo + copyright + links

### Documentation
- [ ] Logo in header (consistent sizing)

- [ ] Color palette: Teal for highlights, grays for text

### Social Media
- [ ] Profile picture: icon.svg (64x64 minimum)

- [ ] Post templates: Consistent color scheme

## 3. Visual Consistency Checklist

### Color Palette
- [ ] Primary: #0f766e (Teal 700) — Trust, security, tech
- [ ] Primary Light: #0d9488 (Teal 600) — Hover states
- [ ] Accent: #06b6d4 (Cyan 500) — Connection nodes, highlights
- [ ] Neutral: #f8fafc (Slate 50) — Light backgrounds
- [ ] Neutral Dark: #1e293b (Slate 800) — Dark backgrounds
- [ ] Text: #0f172a (Slate 900) — Primary text
- [ ] Text Light: #64748b (Slate 500) — Secondary text

### Typography
- [ ] Headings: Geist Sans or Inter (geometric, modern)
- [ ] Body: Inter or system font (readable, accessible)
- [ ] Monospace: JetBrains Mono (code blocks)
- [ ] Font sizes: 12px (small), 14px (body), 16px (large), 20px+ (headings)

### Logo Usage
- [ ] Minimum size: 32px (web), 16px (favicon)
- [ ] Clear space: 1/4 of logo height on all sides
- [ ] Never: Stretch, rotate, change colors (except dark variant)
- [ ] Backgrounds: White/light for primary, dark for dark variant

## 4. SEO Baseline Checklist

- [ ] Meta title: ≤60 characters, includes primary keyword
- [ ] Meta description: 150-160 characters, compelling CTA
- [ ] Canonical URL: Set on all pages
- [ ] Robots meta: index, follow (except admin pages)
- [ ] Viewport: width=device-width, initial-scale=1.0
- [ ] Charset: UTF-8 declared
- [ ] Open Graph: All 6 core tags present (title, description, type, url, image, site_name)
- [ ] Twitter Card: summary_large_image with image
- [ ] JSON-LD: SoftwareApplication + Organization schemas
- [ ] Heading hierarchy: H1 (one per page), H2-H6 logical order
- [ ] Image alt text: Descriptive, includes keywords where relevant
- [ ] Internal links: Descriptive anchor text
- [ ] Mobile-friendly: Responsive design, touch-friendly buttons
- [ ] Page speed: Core Web Vitals optimized (LCP, FID, CLS)

## 5. Social Share Validation Checklist

### Before Publishing
- [ ] Test OG image: 1200x630px, <1MB, PNG/JPG
- [ ] Test Twitter card: Use Twitter Card Validator
- [ ] Test Facebook share: Use Facebook Sharing Debugger
- [ ] Test LinkedIn share: Verify image + description
- [ ] Test WhatsApp share: Ensure title + description visible
- [ ] Mobile preview: Ensure text readable on small screens

### Post-Launch Monitoring
- [ ] Monitor social impressions (Twitter Analytics, etc.)
- [ ] Track click-through rates from social
- [ ] Monitor brand mentions
- [ ] Respond to engagement within 24 hours

## 6. Common SaaS Branding Mistakes to Avoid

❌ **DO NOT:**
- Use generic startup clichés ("Revolutionize", "Disrupt", "Next-gen")
- Change logo colors arbitrarily (breaks brand recognition)
- Use inconsistent naming (Kawa Note vs KawaNote vs Kawa-Note)
- Forget dark mode variants (50% of users use dark mode)
- Ignore mobile responsiveness (60%+ traffic is mobile)
- Use low-quality OG images (impacts social CTR by 30%)
- Forget favicon (impacts brand recall)
- Use outdated meta tags (rel="author", keywords tag)
- Ignore accessibility (WCAG AA minimum)
- Publish without testing social shares

✅ **DO:**
- Keep messaging consistent across all channels
- Test all visual assets on light AND dark backgrounds
- Use data-driven copy (specific benefits, not fluff)
- Monitor SEO metrics monthly
- Update OG images for major campaigns
- Maintain brand guidelines document
- Review branding quarterly for consistency
- Test social shares before publishing
- Use analytics to inform branding decisions
- Document all branding decisions in governance files
