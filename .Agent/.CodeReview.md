# CODE REVIEW RULES

## Checklist Obrigatório
- [ ] O código segue as `.CodeConventions.md`.
- [ ] Não há variáveis ou importações não utilizadas.
- [ ] Hooks são utilizados corretamente e dentro das regras do React.
- [ ] Novos componentes UI estão na pasta correta (`src/components/ui` ou pastas de domínio).
- [ ] Tipagem (JSDoc ou Typescript definitions em jsconfig) está consistente.
- [ ] Erros de lint foram resolvidos (`npm run lint`).

## Critérios de Severidade
- **CRITICAL:** Quebra de build, vazamento de dados, erro de lógica grave. (Bloqueia execução)
- **MAJOR:** Violação de convenções, código redundante, falta de acessibilidade. (Requer correção)
- **MINOR:** Sugestões de estilo, otimizações leves. (Opcional)

## Formato de Relatório de Revisão
Cada execução do `AGENT_EXECUTOR` deve gerar um relatório em `/.Agent/.plans/reports/` seguindo:
1. Resumo das alterações.
2. Lista de arquivos modificados.
3. Status final (Sucesso/Erro).
4. Evidências de testes (se aplicável).
