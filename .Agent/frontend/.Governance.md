# GOVERNANCE — FRONTEND

## Classificação
- **Tipo:** Frontend
- **Framework:** React 18.2.0
- **Build Tool:** Vite 6.1.0
- **Estilização:** Tailwind CSS 3.4.17
- **Escopo:** Interface de usuário, componentes, integração com API

## Relação com Governança Global
Este arquivo **estende e especializa** `/.Agent/.AgentRules.md`, `/.Agent/.CodeConventions.md` e `/.Agent/.CodeReview.md`.

**Nenhuma regra aqui contradiz a governança global.**

## Papéis Permitidos
- **AGENT_AUDITOR:** Auditoria de código, UX, performance, acessibilidade.
- **AGENT_EXECUTOR:** Execução de planos aprovados no escopo `src/`.

## Escopo Permitido
- ✅ Modificar código em `src/`
- ✅ Modificar componentes em `src/components/`
- ✅ Modificar hooks em `src/hooks/`
- ✅ Modificar páginas em `src/pages/`
- ✅ Modificar API client em `src/api/`
- ✅ Modificar estilos em `src/styles/`
- ✅ Modificar `package.json` (dependências frontend)
- ❌ Modificar código em `backend/` (backend)
- ❌ Modificar governança global em `/.Agent/`
- ❌ Modificar regras ou convenções globais

## Critérios de HALT (Bloqueio Imediato)
1. **Build:** Código que não compila com Vite.
2. **Lint:** Código que não passa em `npm run lint`.
3. **Tipo:** Erros de tipagem ou validação Zod.
4. **Teste:** Falha em testes unitários ou de integração.
5. **Performance:** Degradação significativa (LCP, FID, CLS).
6. **Acessibilidade:** Violação de WCAG 2.1 AA.
7. **UX:** Quebra de fluxo de usuário crítico.

## Níveis de Permissão
- **Nível 0:** Observador (default)
- **Nível 1:** Planejador (criar planos)
- **Nível 2:** Executor Limitado (executar planos simples)
- **Nível 3:** Executor Total (executar planos complexos)
- **Nível 4:** Governança (modificar este arquivo)

## Processo de Trabalho
1. **Auditoria:** Verificar código, UX, performance, acessibilidade.
2. **Planejamento:** Criar plano em `/.Agent/.plans/PLAN-<date>-<app>.md`.
3. **Aprovação:** Aguardar confirmação explícita do usuário.
4. **Execução:** Executar plano, gerar relatório em `/.Agent/.plans/reports/`.

## Restrições Adicionais
- Não modificar `src/components/ui/` (Shadcn UI — apenas adicionar novos componentes)
- Não modificar `vite.config.js` sem aprovação explícita.
- Não modificar `tailwind.config.js` sem aprovação explícita.
- Não expor variáveis de ambiente sensíveis em código.

## Sincronização de Documentação
A cada execução, verificar:
- `package.json` vs `/.Agent/frontend/.Stack.md`
- `src/` vs `/.Agent/frontend/.UXRules.md`
- Atualizar se houver divergência.
