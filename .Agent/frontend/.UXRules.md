# UX RULES — FRONTEND

## Princípios de Design
1. **Simplicidade:** Interface limpa e intuitiva.
2. **Consistência:** Padrões visuais e comportamentais uniformes.
3. **Feedback:** Sempre informar ao usuário o resultado de suas ações.
4. **Acessibilidade:** Inclusivo para todos os usuários.
5. **Performance:** Resposta rápida (< 100ms para interações).

---

## Padrões de Componentes

### Botões
- **Primário:** Ação principal (azul)
- **Secundário:** Ação alternativa (cinza)
- **Perigo:** Ação destrutiva (vermelho)
- **Desabilitado:** Estado inativo (opacidade 50%)

### Formulários
- **Labels:** Sempre presentes e associados ao input
- **Validação:** Feedback em tempo real
- **Erros:** Mensagens claras e acionáveis
- **Obrigatório:** Marcar com asterisco (*)

### Modais
- **Título:** Claro e descritivo
- **Conteúdo:** Conciso e focado
- **Ações:** Botões primário e secundário
- **Fechar:** Botão X no canto superior direito

### Notificações
- **Sucesso:** Verde, ícone de check
- **Erro:** Vermelho, ícone de X
- **Aviso:** Amarelo, ícone de alerta
- **Info:** Azul, ícone de informação
- **Duração:** 3-5 segundos (auto-dismiss)

---

## Fluxos de Usuário

### Autenticação
1. Usuário acessa `/login`
2. Preenche email e senha
3. Clica "Entrar"
4. Validação no backend
5. Se sucesso → Redireciona para `/notes`
6. Se erro → Exibe mensagem de erro

### Criar Nota
1. Usuário clica "Nova Nota"
2. Modal abre com formulário
3. Preenche título e conteúdo
4. Clica "Salvar"
5. Validação (Zod)
6. Se sucesso → Nota criada, modal fecha, notificação
7. Se erro → Exibe erro no formulário

### Editar Nota
1. Usuário clica em nota existente
2. Editor abre com conteúdo preenchido
3. Modifica conteúdo
4. Clica "Salvar"
5. Validação
6. Se sucesso → Nota atualizada, notificação
7. Se erro → Exibe erro

### Deletar Nota
1. Usuário clica ícone de delete
2. Modal de confirmação aparece
3. Clica "Confirmar"
4. Nota deletada
5. Notificação de sucesso
6. Lista atualizada

## Responsividade

### Breakpoints
- **Mobile:** < 640px
- **Tablet:** 640px - 1024px
- **Desktop:** > 1024px

### Comportamento
- **Mobile:** Menu hambúrguer, stack vertical
- **Tablet:** Sidebar colapsável, layout adaptado
- **Desktop:** Sidebar fixo, layout completo

## Animações
- **Transição:** 200ms ease-in-out
- **Hover:** Mudança de cor ou escala
- **Loading:** Spinner ou skeleton
- **Erro:** Shake animation

---

## Acessibilidade (WCAG 2.1 AA)

### Cores
- Contraste mínimo 4.5:1 para texto
- Não usar apenas cor para transmitir informação

### Navegação
- Teclado: Tab, Shift+Tab, Enter, Escape
- Focus visible em todos os elementos interativos
- Ordem lógica de tab

### Leitores de Tela
- Labels em formulários
- `aria-label` em ícones
- `aria-describedby` para descrições
- `role` apropriado em elementos customizados

### Imagens
- `alt` text descritivo
- Não usar imagens para transmitir informação crítica

---

## Performance

### Métricas
- **LCP (Largest Contentful Paint):** < 2.5s
- **FID (First Input Delay):** < 100ms
- **CLS (Cumulative Layout Shift):** < 0.1

### Otimizações
- Code splitting com React.lazy
- Image optimization
- Bundle size < 200KB (gzipped)
- Lazy load de componentes não críticos

---

## Temas (Dark/Light)
- **Light:** Fundo branco, texto escuro
- **Dark:** Fundo escuro, texto claro
- **Preferência:** Respeitar `prefers-color-scheme`
- **Toggle:** Botão no header para alternar
