## Tone Guidelines

### DO:
- Be clear and direct (avoid jargon unless necessary)
- Use active voice (we encrypt, not encryption happens)
- Focus on benefits, not features (privacy, not encryption algorithm)
- Be confident but not arrogant (we're good, not the only option)
- Use specific language (not "fast", but "captures in <100ms")
- Show personality (friendly, technical, trustworthy)

### DON'T:
- Use startup clichÃ©s ("revolutionize", "disrupt", "next-gen")
- Make unverifiable claims ("best", "most secure")
- Use passive voice (avoid "is encrypted")
- Overcomplicate (explain like you're talking to a smart friend)
- Use ALL CAPS or excessive punctuation
- Make it about us, make it about the user
